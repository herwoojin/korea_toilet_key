# PRD — Korea Toilet Sharing Service (코리아 토일럿 쉐어링 서비스)

> 제품 요구사항 정의서 (Product Requirements Document)
> 버전: 1.0 | 작성일: 2026-07-19

---

## 1. 제품 개요

### 1.1 한 줄 정의
대한민국 상가/빌딩의 남·여 화장실 위치와 출입 비밀번호를 지도 위에서 확인할 수 있는, 신뢰 기반 크라우드소싱 화장실 공유 웹앱.

### 1.2 배경 및 문제 정의
- 한국의 상가 건물 화장실은 대부분 비밀번호 잠금장치로 관리되며, 비밀번호는 해당 건물의 음식점·주점·분식집 등 입점 상가에서 고객에게만 알려준다.
- 길을 가다 급하게 화장실이 필요한 사람은 상가 사장님에게 물어봐야 하는 심리적 부담이 있고, 상가 입장에서도 비고객 응대가 부담이다.
- 외국인 관광객은 이 문화 자체를 모르기 때문에 더 큰 불편을 겪는다.

### 1.3 제품 비전
"한국에서 살아가는 사람도, 한국을 찾는 외국인도, 생리 현상만큼은 어디서든 정서적 부담 없이 해결할 수 있는 나라" — 한국의 화장실 문화를 세계 최고 수준으로 만드는 공유 플랫폼.

### 1.4 핵심 딜레마와 해결 전략
비밀번호가 무제한 공개되면 ① 누가 썼는지 알 수 없고 ② 청소·관리 부담이 커지고 ③ 상가/건물주에게 책임만 전가된다.
따라서 본 서비스는 **"익명 공개"가 아니라 "기록이 남는 신뢰 기반 열람"** 을 핵심 원칙으로 한다.

| 원칙 | 구현 방식 |
|---|---|
| 열람 = 사용 간주 | 비밀번호를 조회한 순간, 누가/언제/어느 빌딩의 어느 화장실인지 열람 로그가 영구 기록됨 |
| 신원의 실재성 | 구글/카카오 로그인 필수 + (무료 가능 시) 휴대폰 본인인증으로 실명 수준 신뢰 확보 |
| 오너 투명성 | 건물주/상가주는 자기 빌딩 비밀번호를 누가 조회했는지 열람 가능 |
| 합의 기반 진실 추정 | 여러 사용자가 제보한 비밀번호 중 다수·최신 제보 값을 "신뢰도 높은 값"으로 추정 |
| 자발적 공개 | 건물주/상가주가 직접 화장실을 등록·공개하는 "오너 공개" 채널 제공 |

## 2. 목표 사용자 (Persona)

1. **급한 보행자 (일반 사용자)** — 외출 중 갑자기 화장실이 필요한 내국인. GPS로 가장 가까운 사용 가능 화장실과 비밀번호를 30초 안에 찾고 싶다.
2. **외국인 관광객** — 한국의 화장실 비밀번호 문화를 모르는 방문객. 영어/중국어/일본어로 동일한 기능을 쓰고 싶다.
3. **기여자 (제보자)** — 자주 가는 건물의 비밀번호를 알고 있어 제보하고, 커뮤니티 기여로 보상(포인트/등급)을 받고 싶은 사용자.
4. **건물주/상가주 (오너)** — 비고객 응대 부담을 줄이면서도, 누가 화장실을 쓰는지 파악하고 싶은 관리 주체. 원하면 공개하고, 원하면 비공개/삭제를 요구할 수 있어야 한다.

## 3. 핵심 기능 요구사항

### 3.1 지도 & 탐색 (P0)
- FR-01: 앱 진입 시 GPS로 현재 위치를 잡고, 주변 반경(기본 500m)의 등록된 화장실 빌딩을 지도에 마커로 표시한다.
- FR-02: 지도 엔진은 1차 OpenStreetMap(Leaflet), 2차로 정확도 향상을 위해 카카오맵 SDK 전환/병행을 지원한다.
- FR-03: 주소 검색(카카오 주소/좌표 API)으로 특정 빌딩을 찾을 수 있다.
- FR-04: 마커 탭 시 빌딩 상세: 빌딩명, 주소, 남자/여자 화장실 위치(층·방향 설명), 비밀번호 신뢰도, 최근 업데이트 시각.

### 3.2 비밀번호 열람 (P0)
- FR-05: 비밀번호는 기본적으로 가려져 있고, "비밀번호 보기" 버튼을 눌러야 표시된다.
- FR-06: 버튼 클릭 시 열람 로그(사용자 ID, 빌딩 ID, 성별 구분, timestamp, 당시 GPS 좌표)가 즉시 기록된다. 이 기록은 "해당 시각 화장실 사용"으로 간주됨을 사용자에게 사전 고지한다.
- FR-07: 남자 화장실 / 여자 화장실 비밀번호는 별도로 관리·표시한다.
- FR-08: 열람 전 "이용 에티켓 서약" 1회 동의(깨끗이 사용, 상업시설 존중 등).

### 3.3 크라우드소싱 제보 & 신뢰도 합의 (P0)
- FR-09: 로그인 사용자는 누구나 주소지를 지정하고 남/여 화장실 비밀번호를 제보(신규 등록 또는 갱신)할 수 있다.
- FR-10: 동일 빌딩·동일 성별 화장실에 여러 비밀번호가 제보되면, **제보 수 × 최신성 × 제보자 신뢰 점수** 가중 합의 알고리즘으로 대표 비밀번호를 추정한다. (예: 1234를 5명, 5678을 1명이 제보 → 1234를 대표값으로 표시)
- FR-11: 대표 비밀번호에는 신뢰도 등급(높음/보통/낮음)과 근거(제보 n건, 최근 확인 n일 전)를 함께 표시한다.
- FR-12: 열람 후 피드백 — "맞았어요 ✅ / 틀렸어요 ❌" 버튼으로 신뢰도를 실시간 보정한다. "틀렸어요"가 누적되면 신뢰도 하향 및 재제보 유도.

### 3.4 신뢰 강화 장치 (P0~P1) — 요청하신 추가 아이디어 포함
- FR-13 **체류 검증 (P1)**: 사용자가 해당 빌딩 반경 내에 약 10분 이상 머문 것이 확인되면(백그라운드 위치 or 열람 시점 GPS 근접성) "현장 확인 제보"로 가중치를 높인다.
- FR-14 **본인인증 (P1)**: 무료 범위에서 가능한 본인확인 수단(휴대폰 SMS 인증 등)을 추가. 인증 사용자는 배지 표시 + 제보 가중치 상승 + 열람 로그 신뢰성 강화.
- FR-15 **오너 인증 & 오너 공개 (P1)**: 건물주/상가주가 사업자등록번호 또는 관리자 확인 절차로 "오너"로 인증하면 ① 공식 비밀번호 등록(신뢰도 100%) ② 자기 빌딩 열람 로그 조회 ③ 비공개 전환/삭제 요청 권한을 갖는다.
- FR-16 **기여 포인트 이코노미 (P1)**: 제보·피드백·오너 공개 등 기여 활동에 포인트를 부여하고, 신규 사용자는 소량의 무료 열람권으로 시작 → 기여할수록 열람이 자유로워지는 선순환 구조. (무임승차 방지 + 데이터 신선도 유지)
- FR-17 **매너 점수 (P2)**: 열람 로그와 피드백, 신고 이력을 종합한 사용자 신뢰 점수. 낮으면 열람 제한.
- FR-18 **신고 & 즉시 조치 (P0)**: 잘못된 정보, 오너의 비공개 요청, 악용 사례를 신고하면 해당 비밀번호는 즉시 "확인 중" 상태로 가려진다.

### 3.5 다국어 (P0)
- FR-19: 한국어(기본), 영어, 중국어(간체), 일본어 4개 언어 UI를 지원한다. 언어 전환 버튼 상시 노출.
- FR-20: 외국인 온보딩 — "한국 화장실 문화" 안내 페이지(비밀번호 문화 설명, 에티켓)를 4개 언어로 제공.

### 3.6 계정 (P0)
- FR-21: 구글 로그인, 카카오 로그인 2종을 지원한다. 비로그인 상태에서는 지도와 빌딩 위치만 보이고 비밀번호 열람·제보는 불가.

### 3.7 브랜딩 (P0)
- FR-22: 서비스명 "Korea Toilet Sharing Service", 파비콘은 서비스 취지(지도 핀 + 화장실 + 공유)를 담은 이미지로 제작. 페이지 타이틀에 서비스명 반영.

## 4. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 성능 | 첫 지도 렌더 3초 이내, 마커 조회는 지오해시 기반 반경 쿼리로 최적화 |
| 플랫폼 | 모바일 우선 반응형 웹(PWA 지원 고려), 데스크톱 겸용 |
| 보안 | 비밀번호 원문은 로그인+동의 사용자에게만 노출, Firestore Security Rules로 서버 측 통제 |
| 개인정보 | 열람 로그·위치정보 수집 동의(위치기반서비스 고지), 오너에게는 최소한의 식별 정보만 노출(닉네임+인증 여부+시각) |
| 비용 | Firebase 무료 티어(Spark) 내 운영을 1차 목표로 설계 |

## 5. 법적·윤리적 고려사항 (중요)

- 건물주 동의 없는 비밀번호 공개는 분쟁 소지가 있으므로, 서비스 정책상 ① 오너의 비공개/삭제 요청은 즉시 이행 ② 크라우드소싱 정보에는 "커뮤니티 제보 정보이며 정확성·이용 권한을 보장하지 않음" 고지 ③ 오너 공개 채널을 적극 유도하여 장기적으로 "동의 기반 데이터" 비중을 높인다.
- 열람 로그의 오너 제공 범위, 위치정보 처리 방침은 위치정보법·개인정보보호법 관점의 약관/동의 절차를 갖춘다.

## 6. 성공 지표 (KPI)

- 등록 빌딩 수 (목표: 오픈 3개월 내 서울 주요 상권 500개)
- 비밀번호 정확도 (피드백 "맞았어요" 비율 85% 이상)
- 오너 인증 빌딩 비율 (전체의 10% 이상)
- 외국어 UI 사용 비율, 재방문율(WAU/MAU)

## 7. 범위 제외 (Out of Scope, v1)

- 네이티브 앱 (iOS/Android 스토어 배포) — PWA로 대체
- 결제/유료 기능
- 공공화장실 공공데이터 연동 (v2에서 서울시 공공화장실 API 연동 검토)
- 블록체인 기반 분산 저장 (개념적 "공유 장부" 철학만 차용, 구현은 Firestore)
