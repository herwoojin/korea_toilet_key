## 1. 전체 로드맵 (총 약 9.5 작업일, MVP 기준)

```
Week 1                          Week 2
├─ D1   Phase 0 기반 (0.5d)     ├─ D6~7  Phase 4 신뢰 강화 (2d)
├─ D1~2 Phase 1 지도 (1.5d)     ├─ D8    Phase 5 다국어 (1d)
├─ D3~4 Phase 2 인증·열람 (2d)  ├─ D9    Phase 6 마감·배포 (1d)
└─ D5   Phase 3 제보·합의 (1.5d)└─ D10   버퍼 & 실사용 테스트
```

## 2. 마일스톤

| 마일스톤 | 완료 기준 | 목표 |
|---|---|---|
| **M1 — 지도가 뜬다** | GPS 현재 위치 + 주변 시드 빌딩 마커 표시 | D2 |
| **M2 — 열람이 된다** | 로그인 → 서약 → 열람 로그 기록 → 비밀번호 표시 (Rules로 직접 read 차단 검증 통과) | D4 |
| **M3 — 데이터가 자란다** | 제보 → 합의 알고리즘 → 대표 비밀번호 갱신 → 피드백 보정 순환 완성 | D5 |
| **M4 — 신뢰가 쌓인다** | 본인인증 배지, 포인트, 오너 대시보드 동작 | D7 |
| **M5 — 세계가 쓴다** | 4개 언어 완역 + 문화 안내 페이지 | D8 |
| **M6 — 배포** | Vercel 프로덕션 + PWA + E2E 통과 | D9 |

## 4. 콜드스타트 전략 (데이터 없이는 앱이 무용지물)

1. **시드 지역 집중**: 전국이 아니라 유동인구 많은 상권 1곳(예: 강남역/홍대)부터 밀도 있게 채운다.
2. **오너 영업**: 상가 사장님에게 "비고객 화장실 문의 응대가 줄어든다"는 가치로 오너 공개 유도 (오프라인 QR 스티커 제작).
3. **공공화장실 연동(v2)**: 비밀번호 없는 공공화장실을 기본 표시하여 "빈 지도" 문제 완화.
4. **기여 인센티브**: 초기 제보자에게 포인트 2배 이벤트.

## 5. 리스크 관리

| 리스크 | 영향 | 대응 |
|---|---|---|
| 건물주 반발/법적 분쟁 | 높음 | 비공개 요청 즉시 이행(disputed 처리), 커뮤니티 정보 고지 문구, 오너 공개 채널 우선 홍보, 약관 정비 |
| 잘못된 비밀번호로 신뢰 하락 | 높음 | 피드백 루프(맞았어요/틀렸어요) + 신뢰 등급 표시로 기대치 관리 |
| 무임승차 (열람만 하고 기여 안 함) | 중간 | 무료 열람권 3회 + 포인트 이코노미 |
| Firebase 무료 티어 초과 | 중간 | 대표값 캐시로 읽기 절감, 사용량 알림, 초과 시 열람 API 레이트리밋 |
| 카카오 로그인 연동 난이도 | 중간 | OIDC 방식 우선 시도, 실패 시 Custom Token 방식 (2일 버퍼 내 처리) |
| OSM 지도 정확도 불만 | 낮음 | MapProvider 추상화로 카카오맵 전환 준비 완료 |
| 위치정보 규제 (위치기반서비스) | 중간 | 동의 절차 선구현, 사업 확장 시 위치정보사업 신고 요건 검토 |

## 6. 진행 원칙

1. **한 세션 = 한 태스크** — PROMPT.md의 프롬프트를 순서대로 하나씩 실행하고, 완료 확인 후 다음으로.
2. **M2(열람 보안)가 품질 게이트** — Rules 테스트 통과 없이는 Phase 3 진입 금지.
3. 매 Phase 종료 시 모바일 실기기(안드로이드/아이폰)에서 직접 확인.
4. TASK.md 체크박스를 진행 상황 단일 소스로 유지.
